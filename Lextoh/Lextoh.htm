1       Présentation générale de LEXTOH

1.1      Conception globale

LEXTOH est un lemmatiseur multilingue, ça fonctionne comme un web service avec méthodes POST ou GET. Cet outil est un intergiciel. Il est créé basé sur plusieurs lemmatiseurs : xelda, xip, ressource dela(f), nooj, Ariane. La 1ère version est disponible sur http://129.88.64.160/Ci-Hai/Lextroh/

Il existe deux formes dans LEXTOH. Une forme intermédiaire « lemmatix » , permet de gérer/transformer les résultats hétérogènes de chaque lemmatiseur différent. Une forme neutre finale « lemmatix » permet d’uniformaliser les résultats de différents lemmatiseurs.

L’entrée de LEXTOH est un text. Les formats de sorties supportent xml, text, json. Les attributs sont paramétrables par les précisions d’utilisateur. Il est également possible de sortir un graphe comme graphe-Q. ou xml ou json, graphe-q est une structure de donnée.

->pas seulement les lemmatiseurs (morphologie ), et aussi appel des stemmer(词根)

1.2      Architecture globale

Un utilisateur peut accéder le service par une interface web ou par API REST. LEXTOH est composé de trois couches principales : une couche de présentation,  une couche de traitement et une couche de ressource/outil.

Quand le système reçoit la requête de l’utilisateur par la couche de présentation, il les envoie au composant « traitement de requête ». Selon les demandes de l’utilisateur, ce composant fait des demandes de différent lemmatiseur. Une fois ce composant reçoit les résultats de lemmatisation (les formats hétérogènes de lemmatiseur), il analyse les résultats puis crée la première forme : forme intermédiaire « lemmatix ». Dans cette forme, elle uniformalise « les clés » (voir 2.1 Forme « lemmatix »). Pour les différents lemmatiseurs et les différentes langues,  les valeurs de lemmatisations sont différentes. Donc il y a un composant « traitement particulier » pour transmettre le forme intermédiaire vers le forme neutre finale « lemmatix ». Par exemple, on transmet la valeur de part-of-speech « v », « v. » vers « verb ».

Si utilisateur demande un format personnalisé dans sa requête, le système génère sa propre sortie en utilisant le composant « format personnalisé » grâce à la forme neutre finale « lemmatix ».

2       Présentation détaillée de LEXTOH

2.1      Forme « lemmatix »

La forme « lemmatix » est décrite en xml (voir figure 2).  La forme intermédiaire et la forme neutre finale sont  respectées cette forme « lemmatix ».

·  lemmatix : racine d’un fichier

·  syst : systeme de lemmatiseur (xelda, xip, delaf, nooj, ariane)

·  lexeme : token

·  startpos : postion de départ

·  endpos : position finale

·  entry : sense

·  form : forme

·  lemma : lemme

·  type : compound

·  pos : part-of-speach

·  confidence : guessed

·  morpho-tag-list : liste de toutes les analyses de morphologie

·  morpho-tag : ex. SG, P3 etc.

Figure 2 : forme "lemmatix"

Les valeurs de la forme intermédiaire sont des valeurs directes qui ont extrait sur les lemmatiseurs différents. Les valeurs de la forme neutre finale sont des valeurs traitées et uniformalisées des lemmatiseurs différents. Par exemple : <pos>+DET</pos> à <pos>determiant<pos> ou <lemma>tester-v</lemma> à<lemma>tester</lemma>.

2.2      Paramètres de requête au point de vue de l’utilisateur (sur interface et API)

Il y a deux types de paramètre : 1. paramètre obligatoire pour accéder un requête. 2. sortie paramétrable pour générer les sorties différente pour la forme neutre finale.

1.    Paramètres

·      lang : 3 caractères latins

·      langVariant (delaf)

·      tool : Système de lemmatiseur (DELAF, XELDA, XIP, NooJ, Ariane)

·      output : Format de sortie (graphe-Q, txt, JSON, xml)

·      text : entrée

·      debug : 1 ou 0 (pour afficher le composant trace)

·      window(delaf, Ariane?) : Offset des résultats de tokenisation (ex. window=5, text=C'est un test. Le 1er résultat de token = C, donc pour il système va vérifier la lemmatisation de "C", "C'", "C'e", "C'es", "C'est", "C'est ". C’est utile souvent pour les mots composés.)

·      formule : 1 ou 0 (si 1 affichage de cette formulaire, si 0 affichage que des résultat (API REST))

2.     Sortie paramétrable (voir 2.1 Forme « lemmatix »)
